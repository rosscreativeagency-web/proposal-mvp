# Proposal MVP — Codex Master Brief

Build a premium animated interactive proposal website from the provided content and brandbook.

Read these files first:
- proposal-input/content.md
- project-docs/ross-brandbook.pdf

This is an interactive presentation experience, not a marketing website.

Core requirements:
- Use all content from content.md. Do not summarize or skip sections.
- Preserve the proposal structure, dividers, order, and storytelling rhythm.
- Use the ROSS brandbook as the visual foundation.
- Expand the ROSS visual language professionally; do not ignore it.
- RTL-first and Persian-first.
- Use Peyda font.
- Create a cinematic, editorial, animated, interactive proposal experience.
- Use advanced motion, microinteractions, scroll reveals, section transitions, sticky storytelling, parallax, animated typography, and visual rhythm.
- Use many intentional visual placeholders where images, icons, diagrams, product visuals, or mockups are needed.
- Placeholders must look designed, not like empty boxes.
- Each placeholder must clearly say what asset should later replace it.
- Avoid generic SaaS layouts, boring cards, dashboards, and template-like sections.
- Each major section should have a distinct visual rhythm and layout.
- Use available tools: Framer Motion, Lenis, GSAP, React Three Fiber, Drei, Lucide, React Icons, shadcn/ui, SplitType, CountUp, canvas-confetti.

Quality bar:
The result should feel like a high-end creative agency proposal, significantly better than PowerPoint.

Before coding:
1. Analyze content.md.
2. Build a section map.
3. Define a visual/motion direction.
4. Then implement.

Commit and push final changes.
